# 데모 사이트 재설계 설계 문서

날짜: 2026-07-14
브랜치: feature/demo-redesign

## 목표

portfolio-hub 의 데모 19개를 "실제 서비스처럼" 보이도록 재설계한다.
구성(섹션 밀도)과 내용(카피·수치·후기)을 강화하고, 가벼운 인터랙션을 더한다.
데모별 개성 있는 디자인 스타일은 유지한다(포트폴리오가 작업 범위를 보여주는 목적).

## 확정 결정사항

- **스코프**: 전체 19개 데모 페이지
- **품질 기준**: 풍부한 단일 페이지 + 가벼운 인터랙션 (무거운 멀티페이지 라우팅 없음)
- **스타일**: 데모마다 다양성 유지 (공통 컴포넌트로 획일화하지 않음)
- **대중성 + 독특성 균형**: 각 데모는 (1) 해당 업종에서 검증된 대중적 레이아웃·UX 패턴을 기반으로 사용성과 고객 기대를 충족하고, (2) 그 위에 데모를 기억하게 만드는 시그니처 표현 요소를 하나씩 얹는다. 실험이 사용성을 해치지 않게 한다.
- **이미지**: 코드에 주석 마커만 남긴다. 사용자가 GPT로 직접 생성 후 교체. `src` 는 기존 Unsplash URL 유지해 렌더가 깨지지 않게 한다.

## 공통 블루프린트 (모든 데모 최소 기준)

### 콘텐츠
- placeholder식 짧은 문구 제거 → 현실적 카피·가격·수치·기간·주의사항·후기(가상이되 그럴듯)
- 숫자 기반 신뢰지표 포함 (예: 누적 시술 건수, 만족도, 운영 연차)

### 섹션 밀도 (타입별 가감)
Hero → 핵심가치 3~4 → 상세 서비스/상품 → 실적·신뢰지표 → 후기/사례 → FAQ → CTA/문의 → 푸터(사업자정보 형식)

### 가벼운 인터랙션 (데모당 1~2개, 성격에 맞춰 선택)
탭 전환 / 아코디언 FAQ / 폼 제출 상태(접수완료 토글) / 스크롤 리빌 / 카운트업 / 갤러리 필터

### 반응형·접근성 기본
모바일 메뉴, `alt` 텍스트, 충분한 명도 대비

### 대중성 + 독특성 (데모별)
- **대중성 축**: 업종별 검증된 구조를 기본 골격으로 (예: 병원=진료과/의료진/예약, 쇼핑몰=상품그리드/장바구니 유도, 웨딩=인사말/갤러리/오시는길/RSVP). 사용자가 헤매지 않게 한다.
- **독특성 축**: 데모마다 시그니처 표현 1개를 정해 기억에 남게 한다 (예: 과감한 타이포·비대칭 그리드·독특한 컬러 조합·스크롤 연출·질감). 단, 이 표현이 핵심 정보 접근을 방해하지 않아야 한다.
- 각 데모 재설계 시작 시 "대중 골격 + 시그니처 1개"를 먼저 명시하고 진행한다.

## 이미지 마커 규칙 (주석만)

각 `<img>` 바로 위에 아래 형식으로 주석을 남긴다. `src` 는 기존 Unsplash URL 유지.

```html
<!-- AI-IMG: [주제] · [스타일/무드] · [비율 예: 16:9] -->
<!-- prompt: (GPT 이미지 생성에 붙여넣을 영문 프롬프트) -->
<img src="(기존 Unsplash URL 유지)" alt="..." >
```

## 배치 (4단계, 각 배치 후 사용자 확인)

1. **웨딩군 (8)**: classic / modern / botanical / editorial / signature / archive-wedding + vip-party + kids-birthday
2. **의료군 (3)**: dental-clinic / hospital / medical-center
3. **기업·전문직 (4)**: law-firm / architecture / corporate-company / corporate-event
4. **커머스·행사 (4)**: ecommerce / b2b-landing / bio-link / exhibition

파일은 서로 독립적이므로 배치 내 병렬 처리 가능.

## 실행 방식

- A안 채택: 공통 품질 기준 확정 후 카테고리 배치 실행
- 각 데모는 `pages/demos/<name>.vue` 단일 파일 유지 (`definePageMeta({layout:false})` 패턴 유지)
- 기존 스크립트 구조(`ref`, 데이터 배열) 패턴 따름
- index.vue 카드 설명 문구도 각 데모 재설계 후 필요 시 동기화

## 비목표 (YAGNI)

- 공통 Vue 컴포넌트 추출/리팩터링 없음 (다양성 유지와 충돌)
- 백엔드·실제 폼 전송·라우팅 없음 (인터랙션은 프론트 상태 토글까지만)
- 실제 이미지 생성/삽입 없음 (주석 마커만)
- index.vue 전면 재설계 없음 (카드 문구 동기화 정도만)
